NIME 2024 WORKSHOP RESOURCES

WORKSHOP DESCRIPTION

Designing interactive audio applications on Android phones is challenging. Despite the ever growing computational power and feature set of such devices, Android audio app development is notoriously affected by performance as well as workflow issues. This is due to the nature of the Android software ecosystem, that includes partially proprietary firmware and imposes specific development tools. In this hands-on workshop, we introduce NIME attendees to LDSP, an open-source software environment that enables to hack Android phones and program them as embedded Linux audio boards. In the first half of the day, attendees will explore the capabilities of LDSP using phones hacked by the presenters, by trying/extending examples written in C++ and Pd. These span oscillators, samples, effects, sensors and more. In the second half, attendees will be able to explore more advanced features, like spectral and neural processing, or to hack their own phones, using the convenient tools provided by LDSP.

You can read more about LDSP in these two papers.

The next steps can be done in advance of the workshop in order to save time, but we will go through them during the workshop as well.

  • Android NDK

    For Linux and Windows you can find it here. For MacOS, you can find it here or through Homebrew like this.
    Please, not down the path of the NKD as you will need it later. In macOS you can do it with the command

    sudo find / -name 'clang++'

  • CMake

    For all OS, you can find the download here. To install follow the instructions here..
    You can check if it was successfully installed typing in the command line

    cmake --version

  • Ninja Build System

    For Linux you can install via your package manager. For macOS you can install through Homebrew or MacPorts, otherwise follow Windows instructions. For Windows, you can download here

    and then add the path to the binary in your PATH environment variable.
    You can check if it was successfully installed typing in the command line

    ninja --version

  • ADB (android device bridge)

    For Linux you can install via your package manager. For macOS you can install through Homebrew or MacPorts, otherwise follow Windows instructions. For Windows, download the Android SDK, adb will be there

    and then add the path to the binary in your PATH environment variable.
    You can check if it was successfully installed typing in the command line

    adb --version

    To see if ADB can connect to your phone:

    • Activate developer options on your phone
    • Connect your phone to your laptop with a USB cable capable of transfering data (e.g., images), as opposed to one that can only charge the phone
    • Open a shell on your laptop and try to connect/open a shell on your phone with:

      adb shell

      • Your phone will likely ask you for permission before allowing connection, grant it permission.
